--------------------------
Estructura de directorios:
--------------------------

	Ubicación del proyecto (ROOT_PATH): 
		/home/revalidas-exa/repositorios/proyecto-revalidas/Desarrollo
	
	Contenido:
		ROOT_PATH/Data: datos de entrada en formato CSV
		ROOT_PATH/R Scripts: scripts necesarios para la ejecución de algortimos de clasificación en R
		ROOT_PATH/Results: archivos con los resultados generados por los scripts
		ROOT_PATH/Results/analisis-j48: archivos con los distintos resultados analizados para la elección
										de los atributos usados en la clasificación final.
		ROOT_PATH/SQL: Scripts SQL para la creación de la base de datos, tabla, roles, permisos y carga 
					   de datos desde el archivo .csv

----------------------------
Re-generación de resultados:
----------------------------

Para volver a correr los algoritmos de clasificacion con nuevos datos realizar los siguientes pasos:

1.	En la carpeta "/home/revalidas-exa/repositorios/proyecto-revalidas/Desarrollo/Data" 
	se debe generar un archivo con formato CSV, que contenga los datos actualizados de 
	los alumnos, respetando las columnas existentes en el archivo dado 'R2_data_final.csv'.
	

Una forma sencilla de generar este archivo es:
	1.1 Editar el archivo 'Plantilla_Final_Datos_Revalidas_R2.xlsx' incorporando los nuevos datos	
		Columnas esperadas: 
		nro_solicitud,fecha_inscripcion,anio_ingreso,finales_adeudados,cursadas_adeudadas,optativas_adeudadas,adeuda_trab_final_inicio,plan,tutor_r1,tutor_r2,graduado,fecha_graduacion,usable,,,,,,,,,,,
	1.2. Después de cargar los datos en la plantilla, exportar como .csv (Ejemplo 'Ejemplo-exportación-Plantilla_Final_Datos_Revalidas_R2.csv')
	1.3. Realizar los siguientes modificaciones en el arhchivo resultante (Ejemplo:  'Ejemplo-resultado-Plantilla_Final_Datos_Revalidas_R2.csv')
		a. remover el encabezado que contiene los nombres de columna
		b. reemplazar: ',,,,,,,,,,,' por '' (remover las columnas sobrantes que exportacom vacíos)
		c. reemplazar: 'FALSO' por 'FALSE'
		d. reemplazar: 'VERDADERO' por 'TRUE'
	
	1.4. El archivo a ser cargado en la base de datos está indicado en el script "4_copy_data_R2.sql",
		y es por defecto el mencionado anteriormente, puede reemplazarse directamente el contenido 
		para que el script SQL lo levante con el mismo nombre, o bien generar un archivo distinto y 
		editar el script con el nuevo nombre del archivo .csv.

2.	Desde PGAdmin, conectarse a la base de datos "revalidas" y ejecutar el script "4_copy_data_R2.sql"
		usuario BD: revalidas
		contraseña BD: revalidas17

3.	Una vez importados los datos, abrir la terminal y ejecutar los siguientes comandos:
    3.a. Ejecución de J48 (algoritmo seleccionado)	
    revalidas-exa@revalidas-exa-VirtualBox ~ $ cd /home/revalidas-exa/repositorios/proyecto-revalidas/Desarrollo/
    revalidas-exa@revalidas-exa-VirtualBox ~/repositorios/proyecto-revalidas/Desarrollo $ sh ./j48.sh 
        Loading required package: RPostgreSQL                                                                     
        Loading required package: methods                                                                         
        Loading required package: DBI                                                                             
        Loading required package: RWeka  

    3.b. Ejecución de One Rule
    revalidas-exa@revalidas-exa-VirtualBox ~ $ cd /home/revalidas-exa/repositorios/proyecto-revalidas/Desarrollo/
    revalidas-exa@revalidas-exa-VirtualBox ~/repositorios/proyecto-revalidas/Desarrollo $ sh ./OneR.sh

3.	Una vez ejecutados los scripts, los resultados pueden observarse en la carpeta
